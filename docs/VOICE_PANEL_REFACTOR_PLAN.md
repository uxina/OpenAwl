# 语音面板重构方案

## 📊 现状分析

### CMD-001.mp3 技术标准（参考基准）
```
Duration: 18.50秒
Sample Rate: 48000 Hz
Channels: mono (单声道)
Bitrate: 74 kb/s
Format: MP3
```

**音色特点**: 女声、温和、清晰、语速适中

---

## 🔍 当前问题诊断

### 问题 1: 音色不一致
**影响语音**: CMD-016, CMD-017, CMD-019, CMD-021
**问题描述**: 
- CMD-016, CMD-017 使用旧版音色
- CMD-019, CMD-021 音色与 CMD-020 不一致
- 夜间流程中音色切换突兀，影响沉浸感

**影响**: 夜间阶段语音听起来像不同人念的，破坏游戏氛围

### 问题 2: 音量不统一
**问题描述**:
- 部分语音音量过大（如 CMD-044~056 角色引导语音）
- 部分语音音量过小（如 segments/ 片段）
- 数字语音音量与主语音不匹配

**影响**: 玩家需要频繁调节音量，体验不佳

### 问题 3: 停顿节奏不一致
**问题描述**:
- 部分语音句末停顿过长
- 部分语音句末停顿过短
- 语音片段拼接时节奏不连贯

**影响**: 拼接播报时听起来不自然

### 问题 4: 夜间流程设计复杂
**当前设计**:
- 5-7人局：12步流程
- 8-10人局：14步流程
- 包含自动播放步骤和手动步骤混合

**问题**:
- 步骤过多，主持人操作负担重
- 自动播放和手动播放逻辑容易混淆
- 新主持人学习成本高

### 问题 5: 语音分类混乱
**问题描述**:
- CMD-016~022 和 CMD-N01~N15 功能重叠
- 旧版语音和新版语音并存
- 角色引导语音（CMD-044~056）与流程语音混在一起

**影响**: 维护困难，容易选错语音

### 问题 6: 缺少关键语音
**缺失语音**:
- 8-10人局莫德雷德流程语音（CMD-N13, CMD-N14 已存在但未在流程中使用）
- 连续否决警告语音（CMD-083 存在但很少使用）
- 任务执行倒计时语音（CMD-098 存在但很少使用）

## 💡 重构方案

### 方案 A: 保守优化（推荐）

**核心思想**: 保持现有流程框架，优化细节问题

#### A1. 音色统一计划

**第一阶段：重新录制不一致语音**

| 语音ID | 当前问题 | 重录要求 | 优先级 |
|--------|----------|----------|--------|
| CMD-016 | 旧版音色 | 参考 CMD-001 音色 | 高 |
| CMD-017 | 旧版音色 | 参考 CMD-001 音色 | 高 |
| CMD-019 | 音色不一致 | 参考 CMD-001 音色 | 高 |
| CMD-021 | 音色不一致 | 参考 CMD-001 音色 | 高 |

**录制规范**:
```
音色: 女声，温和，清晰
语速: 每分钟约120-130字
音量: -3dB ~ -6dB（避免爆音）
采样率: 48000 Hz（与 CMD-001 一致）
声道: 单声道
格式: MP3，128kbps
句末停顿: 0.5-0.8秒
```

#### A2. 音量标准化

**处理步骤**:
1. 使用 FFmpeg 批量标准化音量
2. 目标音量: -16 LUFS（广播标准）
3. 保留动态范围，避免过度压缩

```bash
# 标准化脚本示例
ffmpeg -i input.mp3 -af "loudnorm=I=-16:TP=-1.5:LRA=11" -ar 48000 output.mp3
```

#### A3. 夜间流程简化 ✅ 已完成

**优化结果**: 从12-14步简化到**6步**

**最终方案**:

##### 5-7人局夜间流程（6步）

| 步骤 | 语音ID | 说明 |
|------|--------|------|
| 1 | CMD-N01 | 夜间开始-闭眼 |
| 2 | CMD-N02 | 坏人睁眼互认（除奥伯伦） |
| 3 | CMD-N03 | 坏人闭眼，放下大拇指 |
| 4 | CMD-N05 | 梅林睁眼观察（所有坏人竖拇指，含奥伯伦） |
| 5 | CMD-N09 | 派西维尔睁眼观察 |
| 6 | CMD-N12 | 所有人睁眼 |

##### 8-10人局夜间流程（6步）

| 步骤 | 语音ID | 说明 |
|------|--------|------|
| 1 | CMD-N01 | 夜间开始-闭眼 |
| 2 | CMD-N02 | 坏人睁眼互认（除奥伯伦，含莫德雷德） |
| 3 | CMD-N03 | 坏人闭眼，放下大拇指 |
| 4 | CMD-N05-M | 梅林睁眼观察（除莫德雷德外坏人竖拇指） |
| 5 | CMD-N09 | 派西维尔睁眼观察 |
| 6 | CMD-N12 | 所有人睁眼 |

**关键优化**:
- 删除了莫德雷德单独睁眼步骤（合并到坏人互认）
- 删除了大拇指提示单独步骤（合并到主语音）
- 5-7人局和8-10人局只差1条语音（CMD-N05 vs CMD-N05-M）
- 角色视野规则更新：奥伯伦能被梅林看到，莫德雷德不能被梅林看到

**语音清单**: 共7条新语音
- 通用：CMD-N01, CMD-N02, CMD-N03, CMD-N09, CMD-N12（5条）
- 5-7人局：CMD-N05（1条）
- 8-10人局：CMD-N05-M（1条）

### 方案 B: 激进重构

**核心思想**: 重新设计语音面板架构

#### B1. 场景化语音设计

不按游戏阶段分类，按使用场景分类：

```
scenes/
├── scene-01-welcome/       # 欢迎场景
├── scene-02-night-5-7/     # 5-7人夜间场景
├── scene-03-night-8-10/    # 8-10人夜间场景
├── scene-04-day-discuss/   # 白天讨论场景
├── scene-05-vote/          # 投票场景
├── scene-06-mission/       # 任务场景
├── scene-07-assassin/      # 刺杀场景
└── scene-08-end/           # 结束场景
```

每个场景包含完整的语音序列，一键播放。

#### B2. 智能播放模式

**模式1: 手动模式（默认）**
- 主持人控制每一步
- 适合新手和正式比赛

**模式2: 自动模式**
- 自动播放完整流程
- 可设置每步等待时间
- 适合熟练主持人和快速游戏

**模式3: 混合模式**
- 关键步骤手动控制
- 辅助步骤自动播放
- 平衡效率和体验

#### B3. 语音拼接优化

**当前问题**: 数字和片段拼接生硬
**优化方案**:
```javascript
// 使用 Web Audio API 实现平滑过渡
const audioContext = new AudioContext();

// 淡入淡出效果
function playWithFade(audioBuffer, fadeIn = 0.1, fadeOut = 0.1) {
    const source = audioContext.createBufferSource();
    const gainNode = audioContext.createGain();
    
    source.buffer = audioBuffer;
    source.connect(gainNode);
    gainNode.connect(audioContext.destination);
    
    // 淡入
    gainNode.gain.setValueAtTime(0, audioContext.currentTime);
    gainNode.gain.linearRampToValueAtTime(1, audioContext.currentTime + fadeIn);
    
    // 淡出
    const duration = audioBuffer.duration;
    gainNode.gain.setValueAtTime(1, audioContext.currentTime + duration - fadeOut);
    gainNode.gain.linearRampToValueAtTime(0, audioContext.currentTime + duration);
    
    source.start();
}
```
